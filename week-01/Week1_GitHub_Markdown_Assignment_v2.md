# Week 1 Assignment

## GitHub Setup & Your First Markdown File

------------------------------------------------------------------------

## Purpose

This assignment introduces two essential tools for this course:

-   **GitHub** (for collaboration and professional workflow)
-   **Markdown** (for clear, structured communication)

By the end of this assignment, you will: - Create or access a GitHub
account\
- Create a repository\
- Write and publish a properly formatted Markdown document\
- Connect with classmates on GitHub

These are foundational skills for data science and technical
communication.

------------------------------------------------------------------------

## Part 1 --- Create Your GitHub Account (If Needed)

If you already have a GitHub account, move to Part 2.

1.  Go to: https://github.com\
2.  Click **Sign Up**\
3.  Choose:
    -   A professional username\
    -   A valid email address\
    -   A secure password\
4.  Verify your email.

Use a username you would feel comfortable sharing on a résumé.

------------------------------------------------------------------------

## Part 2 --- Create (or Locate) Your Repository

If you created a repository in class, log in and navigate to it (for
example: `Data110`).

If you do **not** have a repository yet:

1.  Click the **"+"** icon (top right).
2.  Select **New repository**.
3.  Name it: `Data201_YourName`
4.  Set visibility to **Public**
5.  Click **Create repository**

------------------------------------------------------------------------

## Part 3 --- Create Your Markdown File

Inside your repository:

1.  Click **Add file**
2.  Select **Create new file**
3.  Name the file:

```{=html}
<!-- -->
```
    Markdown-Exercise.md

------------------------------------------------------------------------

## Part 4 --- Markdown Practice Requirements

In `Markdown-Exercise.md`, include all of the following elements.

### 1. Author Section

At the top of your document:

-   Your name\
-   Today's date

------------------------------------------------------------------------

### 2. Headers

Use at least **three different header levels** to organize your content.

------------------------------------------------------------------------

### 3. Emphasis

Demonstrate: - **Bold text** - *Italic text*

------------------------------------------------------------------------

### 4. Lists

Include: - One ordered list\
- One unordered list

------------------------------------------------------------------------

### 5. Links

Add at least one hyperlink to a website of your choice.

------------------------------------------------------------------------

### 6. Images

Insert one image using an online image URL.\
(You may use a personal interest, hobby, favorite place, etc.)

------------------------------------------------------------------------

### 7. Blockquote

Include one meaningful quote using Markdown blockquote syntax.

------------------------------------------------------------------------

### 8. Inline Code & Code Block

Demonstrate: - Inline code using single backticks\
- A multi-line code block using triple backticks

------------------------------------------------------------------------

### 9. Table

Create a simple table (example topics): - Programming languages
comparison\
- Favorite hobbies\
- Tools you want to learn

------------------------------------------------------------------------

### 10. Task List

Create a task list with at least three items, and check off at least one
item.

------------------------------------------------------------------------

### 11. Footnote

Add at least one footnote explaining something further.

------------------------------------------------------------------------

## Part 5 --- Connect with Classmates

After creating your repository:

-   Follow **at least 3 of your classmates** on GitHub.
-   Explore their repositories.
-   Make sure your own repository is public so others can view it.

This helps build a collaborative learning environment and mirrors
real-world professional networking.

------------------------------------------------------------------------

## Part 6 --- Commit Your Work

When finished:

1.  Scroll to the bottom of the page.

2.  Write a meaningful commit message\
    Example:

        Add Markdown-Exercise.md with required formatting elements

3.  Select **Commit directly to the main branch**

4.  Click **Commit new file**

------------------------------------------------------------------------

## Submission Instructions

1.  Open your `Markdown-Exercise.md` file.
2.  Copy the URL from your browser.
3.  Paste the link into the file named **Github_link** in our MS Teams
    channel.

Make sure the link opens publicly without login issues.

------------------------------------------------------------------------

## Evaluation Criteria

Your grade will be based on:

-   Correct use of Markdown syntax\
-   Inclusion of all required elements\
-   Organization and clarity\
-   Effort and creativity\
-   Following at least 3 classmates on GitHub

This is not about writing something long. It is about writing something
structured and correct.
